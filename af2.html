Procesos, Hilos y Concurrencia en Ubuntu 24.04

Problemas Centrales de la Concurrencia

La concurrencia surge cuando dos o más procesos o hilos se ejecutan de manera simultánea y comparten recursos del sistema. Esto permite un mejor aprovechamiento del procesador pero introduce problemas complejos que deben ser controlados.

Condiciones de Carrera (Race Conditions)

Una condición de carrera ocurre cuando el resultado final de una ejecución depende del orden en que se intercalan las instrucciones de los procesos. Este orden depende del planificador del sistema operativo y de factores como interrupciones o tiempos de ejecución.

El problema aparece cuando varios procesos acceden y modifican un recurso compartido al mismo tiempo, como un archivo o un registro en memoria.

Comando útil en Ubuntu para detectar condiciones de carrera:

$ strace -p PID

Secciones Críticas

Una sección crítica es la parte del código donde un proceso accede a un recurso compartido que no puede ser utilizado simultáneamente por otro proceso sin provocar inconsistencias.

Ejemplos de operaciones en sección crítica:

  • Actualizar una variable compartida.
  • Escribir en un archivo común.
  • Modificar estructuras del sistema operativo.

Requerimientos para una Solución

  • Exclusión Mutua: Garantiza que solo un proceso puede estar dentro de su sección crítica en un momento dado.
  • Progreso: Si ningún proceso está ejecutando su sección crítica, la decisión de quién entra no puede posponerse indefinidamente.
  • Espera Limitada: Asegura que existe un límite de tiempo para que un proceso acceda para que no haya una inanición (starvation).

Comandos en Ubuntu para gestionar procesos y concurrencia

$ ps aux # Listar procesos
$ htop # Monitoreo interactivo
# pthread_create # Crear hilos (programación en C)
$ kill -9 PID # Terminar un proceso
$ nice -n 10 ./programa # Ejecutar con prioridad baja
$ taskset -c 0 ./programa # Fijar proceso a un núcleo específico
