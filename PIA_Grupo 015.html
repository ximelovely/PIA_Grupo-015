Sistemas Operativos

Explorando el corazón de tu computadora con elegancia y precisión, aplicando cada concepto en Ubuntu 24.04 LTS.

1. Definición y Componentes

Un Sistema Operativo (SO) es el software principal que actúa como un puente elegante entre tú (la usuaria), el hardware de tu computadora y las aplicaciones. Si pensamos en Ubuntu 24.04 LTS, este puente está diseñado para ser seguro, de código abierto y sumamente estable.

Componentes y sus Funciones en Ubuntu 24.04

Núcleo (Kernel Linux 6.8)

Es el corazón del sistema. En Ubuntu 24.04, gestiona la memoria, los procesos y el hardware. Opera en un área protegida llamada Espacio Kernel.

Llamadas al Sistema (Syscalls)

La forma educada en la que las aplicaciones piden favores al kernel (ej. leer un archivo). Actúan como la frontera entre el Espacio de Usuario y el Kernel.

Interfaz de Usuario

La cara visible. En Ubuntu 24.04 usamos GNOME 46 (entorno gráfico) y Bash (nuestra elegante terminal) para interactuar cómodamente.

Espacio de Usuario vs. Espacio Kernel: Tu navegador web o editor de texto viven en el Espacio de Usuario. No pueden tocar el hardware directamente. Cuando necesitan guardar un documento, hacen una llamada al sistema para pedirle al Espacio Kernel que lo haga por ellos de forma segura.

Manejo de Recursos y Partes de Gestión

El núcleo se divide en gestores especializados que mantienen todo en armonía:

  • Gestor de Procesos (Planificador): Decide qué aplicación usa el procesador (CPU) en cada milisegundo.
  • Gestor de Memoria: Reparte la RAM. Si abres muchas pestañas, decide qué mantener en memoria rápida.
  • Gestor de E/S (Entrada/Salida): Administra teclados, pantallas y red.
  • Gestor de Archivos: Organiza tus datos en el disco duro.

Herramientas en Ubuntu para manejo de recursos:

# Ver el uso de CPU y RAM en tiempo real en la terminal
$ top (o el más visual e interactivo: $ htop)

# Para herramientas gráficas, puedes buscar en el menú:
"Monitor del Sistema" (System Monitor)

Glosario Esencial de Ubuntu

Kernel: El núcleo Linux (versión 6.8 en Ubuntu 24.04).
Shell: Intérprete de comandos (por defecto Bash).
Daemon: Procesos silenciosos en segundo plano.
systemd / init: El primer proceso que arranca y gestiona a los demás.
Wayland: El protocolo moderno que dibuja las ventanas en tu pantalla.
GNOME: El entorno de escritorio visual.
apt / snap: Herramientas para instalar aplicaciones.
GRUB: El menú inicial que carga el sistema al encender.

Categorías y Comparación de SO

Los sistemas operativos se pueden clasificar. Por su arquitectura, Ubuntu (Linux) es un núcleo monolítico (todo el núcleo trabaja en el mismo espacio de memoria para ser muy rápido). Por usuarios, es un sistema multiusuario real (varias personas pueden conectarse al mismo tiempo sin interferir).

Ubuntu 24.04 vs Otros SO:

  • Windows 11: Núcleo híbrido, enfoque en escritorio y gaming. Ubuntu es superior en desarrollo, servidores y personalización libre.
  • macOS Sonoma: Basado en Unix, elegante pero cerrado a hardware Apple. Ubuntu te da una experiencia Unix libre en cualquier PC.
  • Android 14 / ChromeOS: Utilizan el kernel de Linux debajo, pero están súper especializados (móviles y nube web).
  • Fedora / Debian: Hermanos de Linux. Debian es el padre conservador de Ubuntu. Fedora usa software más experimental.

2. Procesos e Hilos

Un proceso es simplemente un programa en ejecución. Cuando abres Firefox en Ubuntu, nace un proceso.

Estados de un Proceso y Transiciones

En su ciclo de vida, un proceso viaja por diferentes estados:

  • Nuevo: El sistema está preparando la aplicación.
  • Listo (Ready): En la fila de espera, listo para usar el procesador.
  • En Ejecución (Running): El procesador está ejecutando sus instrucciones.
  • Bloqueado (Sleeping/Waiting): Pausado esperando algo (ej. esperando que el usuario escriba algo).
  • Terminado (Zombie/Dead): Finalizó, pero el SO está limpiando su rastro.
# Ver todos los procesos y su estado en Ubuntu (columna STAT)
$ ps aux

Control y Gestión de Procesos

Identificadores

Cada proceso recibe un PID (Process ID). Su creador es el PPID (Parent PID).

Prioridades

Podemos ser amables ("nice") con el sistema. Un proceso con nice alto cede el paso a otros.

# Cambiar prioridad (ej. hacerlo menos prioritario)
$ renice +10 PID

Señales

Formas de comunicarnos con el proceso. La señal más famosa es terminarlo.

# Cerrar amablemente (SIGTERM)
$ kill PID
# Forzar cierre inmediato (SIGKILL)
$ kill -9 PID
Multiprocesamiento Simétrico (SMP) y Núcleos: Ubuntu 24.04 soporta SMP a la perfección. Esto significa que si tu computadora tiene 8 núcleos, el kernel de Linux (monolítico, a diferencia de sistemas de micro kernel que separan sus funciones básicas en procesos lentos) reparte los procesos de forma equitativa entre todos los núcleos simultáneamente.

Hilos y Concurrencia

Un proceso puede dividirse en tareas más pequeñas llamadas hilos (threads). En Ubuntu, la librería clásica para desarrolladores es `pthreads` (POSIX threads). Los hilos comparten la memoria del proceso padre, lo que los hace más ligeros de crear.

Tipos de Concurrencia en la Terminal:

# Paralelismo real: ocurre solo si tienes múltiples núcleos.
# Concurrencia: lanzar una tarea en "background" (segundo plano) para seguir usando la terminal.
$ comando_largo &
$ jobs # Ver tareas en fondo
$ fg # Traer al frente (foreground)
$ bg # Enviar al fondo (background)

Modelos, Ventajas e Interacciones

Al programar para Ubuntu, podemos usar varios modelos: multiproceso (procesos aislados, más seguro), multihilo (comparten memoria, más rápido), o reactivo/actores (responden a eventos). La ventaja principal es una respuesta inmediata (responsiveness) y mejor uso de recursos.

La convivencia entre procesos

Los procesos pueden ser independientes o cooperativos. Cuando cooperan, surgen interacciones delicadas:

  • Competencia por recursos: Dos apps queriendo imprimir al mismo tiempo.
  • Sincronización y Exclusión Mutua: Mecanismos (como semáforos) para que solo uno modifique un archivo a la vez.
  • Deadlock (Abrazo mortal): Un error fatal donde el proceso A espera al B, y el B espera al A. ¡Ambos se quedan congelados infinitamente!

3. Memoria y Sistemas de Archivos

La forma en que Ubuntu recuerda y organiza nuestra información es vital para su elegancia y velocidad.

Tipos de Memoria y Sistemas de Archivos en Ubuntu 24.04

Tipos de Memoria

  • RAM: Memoria principal, ultrarrápida pero volátil.
  • Swap (Intercambio): Espacio en tu disco duro que Ubuntu usa como "RAM de emergencia" (ahora típicamente un archivo `/swapfile`).
  • Caché y Buffer: RAM sobrante que Ubuntu usa inteligentemente para guardar archivos recientes y acelerar el sistema.

Sistemas de Archivos

  • ext4: El estándar confiable por defecto en Ubuntu.
  • Btrfs / XFS: Para usuarios avanzados o servidores (excelentes para instantáneas y grandes volúmenes).
  • NTFS / FAT: Sistemas de Windows. Ubuntu los lee perfectamente si conectas un USB.

Control de Espacios en Memoria

¿Cómo sabemos cuánta memoria nos queda? Usa estos comandos:

# Ver memoria RAM de forma legible (Human readable)
$ free -h

# Leer directamente de los archivos del kernel
$ cat /proc/meminfo
$ cat /proc/swaps

# Estadísticas de memoria virtual
$ vmstat 2

También puedes abrir la aplicación gráfica Monitor del Sistema y revisar la pestaña "Recursos".

Problemas Frecuentes y Soluciones Prácticas

  • Fragmentación: Espacios vacíos desperdiciados. En Linux (ext4) la fragmentación externa es casi inexistente gracias a cómo guarda los bloques, ¡no necesitas "desfragmentar" como en sistemas antiguos!
  • Fugas de Memoria (Memory Leaks): Un programa mal programado que no devuelve la RAM que pidió. Solución: identificarlo con top y cerrarlo con kill.
  • Sobrecarga de Caché: A veces la RAM parece llena, pero es pura caché. Ubuntu la libera sola, pero si necesitas limpiarla manualmente (como root):
    # echo 3 > /proc/sys/vm/drop_caches
  • Swapping excesivo (Thrashing): Cuando tu PC se pone lentísima porque no tiene RAM y usa el disco duro constantemente.
    Solución elegante: Ajustar la swappiness (qué tanto depende Ubuntu del swap).
    $ sysctl vm.swappiness=10 (Le dice a Ubuntu que use la swap solo en extrema necesidad).

4. Seguridad en Ubuntu

Aunque Linux tiene una arquitectura sumamente segura por diseño (gracias a sus permisos estrictos), no es invulnerable. Mantener tu entorno protegido requiere buenas prácticas.

Amenazas, Intrusos y Autenticación

Amenazas

Los virus tradicionales de Windows no funcionan aquí. Sin embargo, Linux es objetivo de Trojans, Malware para IoT (ej. botnets como Mirai), mineros de criptomonedas (ej. Linux.Lady) y Ransomware enfocado a secuestrar servidores web.

Tipos de Intrusos

Desde Script Kiddies (aficionados usando herramientas copiadas) hasta atacantes sofisticados o Insider Threats (empleados maliciosos). Suelen entrar por servicios mal configurados.

Autenticaciones

• Contraseña local (cifrada).
• SSH con llaves criptográficas (infinitamente más seguro).
• 2FA (Autenticación de 2 factores) vía libpam-google-authenticator.
• Biometría (Huella dactilar vía fprint, soportado nativamente en muchos laptops modernos).

Niveles de Seguridad y Prevención

Perfiles de Protección:
Usuario Doméstico: Activar el cortafuegos UFW, dejar actualizaciones automáticas e instalar aplicaciones vía snap (que vienen en un "sandbox" o entorno aislado gracias a AppArmor).
Red Empresarial: Auditar con auditd, bloquear ataques de fuerza bruta con Fail2ban.
Servidor: Deshabilitar el acceso del usuario "root" por SSH, usar puertos no estándar, requerir siempre llaves SSH.

Análisis de Problemas y Monitoreo

# Ver puertos abiertos a internet (¿hay algún servicio colado?)
$ ss -tuln

# Revisar procesos en forma de árbol para ver actividades raras
$ ps auxf

# Revisar intentos de inicio de sesión fallidos
$ sudo cat /var/log/auth.log

Prevención de Desastres y Manejo de Riesgos

La regla de oro de la protección de la información es la regla 3-2-1: Mantén 3 copias de tus datos, en 2 medios diferentes, con 1 copia fuera de sitio (nube o lugar físico distinto).

  • Herramientas: Usa Deja Dup (Copias de Seguridad en entorno gráfico) para tus carpetas de usuario, y Timeshift para hacer instantáneas del sistema completo (ideal si usas Btrfs o rsync).
  • Administración de Riesgos: Mantén tu sistema al día con un elegante sudo apt update && sudo apt upgrade.

Seguridad a nivel Hardware y Archivos

Si te roban el portátil, una simple contraseña no detendrá al ladrón si saca el disco duro. La solución es el Cifrado de disco completo (LUKS), el cual puedes activar muy fácilmente durante la instalación de Ubuntu 24.04.

A nivel de software, domina los permisos. Utiliza chmod (cambiar permisos de lectura/escritura) y chown (cambiar el dueño). Para casos complejos, usa ACL (Listas de Control de Acceso).

Herramientas de auditoría extra:

  • Gufw: Interfaz gráfica hermosa y sencilla para tu cortafuegos.
  • ClamAV: Un antivirus para escanear archivos (muy útil si compartes archivos con sistemas Windows).
  • Rkhunter / Lynis: Herramientas avanzadas para auditar tu sistema y buscar rootkits (programas que se esconden en el kernel).
