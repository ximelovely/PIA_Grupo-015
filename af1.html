Definición y Componentes del Sistema Operativo - Ubuntu 24.04

Introducción

Ubuntu es un sistema operativo de código abierto basado en la distribución Debian de GNU/Linux. Fue lanzado en 2004 por la empresa Canonical, liderada por Mark Shuttleworth, con el objetivo de hacer que Linux fuera accesible y fácil de usar para el usuario promedio sin perder la potencia que requieren los desarrolladores y administradores de sistemas. La filosofía de Ubuntu se basa en el término africano homónimo, que se traduce aproximadamente como "humanidad hacia otros". Ubuntu se destaca por su ciclo de lanzamientos predecible (versiones regulares cada seis meses y versiones LTS o Long Term Support cada dos años), su robusta seguridad, su extensa comunidad de soporte y su naturaleza gratuita.

Componentes del Sistema Operativo y sus funciones

1. El Núcleo o Kernel

El kernel de Ubuntu es de tipo monolítico modular. Maneja funciones críticas y carga "módulos" bajo demanda.

  • Gestión de Interrupciones: Maneja señales del hardware y decide qué proceso atenderlas.
  • Abstracción de Hardware: Interfaz uniforme para programas.

2. Gestión de Procesos

Ubuntu utiliza el Completely Fair Scheduler (CFS) para maximizar la eficiencia.

  • Creación y Terminación: Llamadas como fork() y exec().
  • Sincronización: Semáforos y bloqueos (mutex).
  • Estados: "Ejecutándose", "Durmiendo" o "Zombi".

3. Gestión de Memoria Principal

Modelo de Memoria Virtual mediante paginación (4KB). Asegura protección entre programas y gestiona el Swap.

4. Gestión del Sistema de Archivos

Uso predeterminado de ext4 con jerarquía unificada desde la raíz /, soportado por Inodos y Journaling para recuperación.

5. Gestión de Entrada/Salida (I/O)

Drivers y áreas de Buffering/Spooling para compensar velocidades entre CPU y periféricos.

6. Sistema de Seguridad

Módulos como AppArmor (MAC) y Control de Acceso Discrecional (DAC) para permisos de usuarios.

Función y características en el manejo de recursos

  • Gestión del Procesador (CPU): Ubuntu utiliza CFS con planificación basada en tiempo de ejecución usando un árbol rojo-negro. Permite ajustar prioridades mediante valores "nice".
  • Gestión de la Memoria (RAM): Memoria virtual, paginación bajo demanda, OOM Killer que sacrifica procesos cuando la memoria se agota.
  • Gestión de Dispositivos (I/O): Sistemas de archivos virtuales (VFS), manejo de interrupciones, DMA para transferencias directas.
  • Almacenamiento Secundario: Planificadores de disco (Deadline, BFQ, Kyber), caché de escritura, journaling con ext4.

Clasificación según el número de usuarios

Multiusuario: Ubuntu es estrictamente multiusuario. Varios usuarios pueden iniciar sesión al mismo tiempo (local o vía SSH), con aislamiento de archivos en /home/, y solo el superusuario (sudo) puede hacer cambios globales.

Partes de gestión de un sistema operativo

  • Gestión de Arranque (systemd): Primer proceso (PID 1). Usa systemctl.
  • Gestión de Paquetes: APT para .deb, Snap para contenedores.
  • Gestión de Redes: NetworkManager (escritorio) y Netplan (servidores).
  • Autenticación: PAM (Módulos de autenticación conectables).
  • Volúmenes Lógicos (LVM): Agrupa y expande particiones.
  • Registros: journald y Syslog.
