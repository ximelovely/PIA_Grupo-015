AF4

Seguridad en Sistemas Operativos - Implementación en Ubuntu

1. Identificación de Amenazas

Comandos utilizados en Ubuntu para identificar y auditar el estado del sistema:

$ top # Monitoreo de procesos en tiempo real
$ netstat -tulnp # Ver puertos abiertos y servicios escuchando
$ who # Usuarios conectados al sistema
$ last # Historial de inicios de sesión
$ journalctl -xe # Revisar logs del sistema

2. Seguridad a nivel sistema

$ sudo adduser usuario_seguro
$ sudo chmod 700 /home/usuario_seguro

3. Firewall (UFW)

$ sudo ufw enable
$ sudo ufw allow 22/tcp
$ sudo ufw allow 80/tcp
$ sudo ufw allow 443/tcp
$ sudo ufw status verbose

4. Protección contra ataques de fuerza bruta (Fail2ban)

$ sudo apt update
$ sudo apt install fail2ban
$ sudo systemctl start fail2ban
$ sudo systemctl enable fail2ban
$ sudo fail2ban-client status sshd

5. Backups y recuperación

# Copia simple
$ cp -r /home/usuario /home/backup_usuario

# Copia incremental con rsync
$ rsync -av --delete /home/usuario /media/backup/

# Respaldo con compresión
$ tar -czvf backup_$(date +%Y%m%d).tar.gz /home/usuario

6. Cifrado de archivos y discos

# Cifrado de archivo individual con GPG
$ gpg -c archivo_sensible.txt

# Descifrar
$ gpg archivo_sensible.txt.gpg

# Verificar si LUKS (cifrado de disco) está activo:
$ lsblk -o NAME,FSTYPE,MOUNTPOINT | grep crypto

Herramientas Adicionales

Monitoreo y Auditoría

Lynis: sudo lynis audit system

Tcpdump: sudo tcpdump -i eth0 -c 100 -w cap.pcap

Auditd: Auditoría detallada del sistema.

Protección Especializada

ClamAV: Antivirus para Linux.

Rkhunter: Detección de rootkits.

AppArmor: Control de acceso obligatorio (MAC).

10. Conclusión

La implementación de herramientas de seguridad en Ubuntu permite aplicar de manera práctica los conceptos teóricos de protección en sistemas operativos. Un sistema Ubuntu bien configurado (con firewall, fail2ban, backups cifrados y monitoreo) es altamente resistente a amenazas externas.
